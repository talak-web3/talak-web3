This guide covers installing Talak Web3 and setting up your first project.

## Choose Your Setup

<Tabs items={["CLI (Recommended)", "Manual"]} defaultIndex={0}>
  <Tab value="CLI (Recommended)">
    The CLI scaffolds a project with the recommended structure:

    ```bash
    npx talak-web3 init my-dapp --template nextjs
    cd my-dapp
    pnpm install
    pnpm dev
    ```

    | Template | Description |
    |----------|-------------|
    | `nextjs` | Next.js App Router with TypeScript |
    | `hono` | Hono server with minimal setup |
    | `express` | Express.js server |

    The CLI handles environment variables and initial configuration.

  </Tab>

  <Tab value="Manual">

    Install the core package or individual packages:

    ```package-install
    talak-web3
    ```

    <details>
    <summary>Individual packages</summary>

    ```package-install
    @talak-web3/auth
    ```
    ```package-install
    @talak-web3/rpc
    ```
    ```package-install
    @talak-web3/client
    ```
    </details>

    Create a `.env` file in your project root:

    ```txt title=".env"

    TALAK_SECRET=your-secret-key-at-least-32-characters

    ```

    <Callout type="info">
      For secret rotation, use `TALAK_SECRETS` (plural) as a comma-separated list of secrets.
    </Callout>

  </Tab>
</Tabs>

<Tabs items={["Next.js App Router", "Next.js Pages Router", "Hono", "Express"]} defaultIndex={0}>
  <Tab value="Next.js App Router">
    Create `talak.config.ts`:

    ```ts title="talak.config.ts"
    import { talakWeb3 } from "talak-web3";
    import { nextCookies } from "talak-web3/nextjs";
    import {
      InMemoryNonceStore,
      InMemoryRefreshStore,
      InMemoryRevocationStore,
    } from "@talak-web3/auth";

    export const app = talakWeb3({
      chains: [
        {
          id: 1,
          name: "Ethereum",
          rpcUrls: ["https://cloudflare-eth.com"],
          nativeCurrency: { name: "Ether", symbol: "ETH", decimals: 18 },
        },
      ],
      auth: {
        nonceStore: new InMemoryNonceStore(),
        refreshStore: new InMemoryRefreshStore(),
        revocationStore: new InMemoryRevocationStore(),
      },
      plugins: [nextCookies()],
    });
    ```

    Add the API route:

    ```ts title="app/api/auth/[...talak]/route.ts"
    import { toNextJsHandler } from "talak-web3/nextjs";
    import { app } from "@/talak.config";

    const handler = toNextJsHandler(app);

    export const GET = handler.GET;
    export const POST = handler.POST;
    export const PUT = handler.PUT;
    export const PATCH = handler.PATCH;
    export const DELETE = handler.DELETE;
    ```

    See the [Next.js integration guide](/docs/nextjs) for client usage, Server Components, and middleware.

  </Tab>

  <Tab value="Hono">
    Create `talak.config.ts` (same as Next.js, without `nextCookies`), then mount `app.handler`:

    ```ts title="src/index.ts"
    import { Hono } from "hono";
    import { app } from "./talak.config.js";

    const server = new Hono();
    server.all("/api/auth/*", async (c) => app.handler(c.req.raw));
    ```

  </Tab>

  <Tab value="Express">
    Create `talak.config.ts`, then mount `app.handler`:

    ```ts title="server.ts"
    import express from "express";
    import { app } from "./talak.config.js";

    const server = express();
    server.all("/api/auth/*", async (req, res) => {
      const response = await app.handler(
        new Request(`http://${req.headers.host}${req.url}`, {
          method: req.method,
          headers: req.headers as HeadersInit,
          body: req.method !== "GET" && req.method !== "HEAD" ? req : undefined,
        }),
      );
      res.status(response.status);
      response.headers.forEach((value, key) => res.setHeader(key, value));
      res.send(Buffer.from(await response.arrayBuffer()));
    });
    ```

  </Tab>
</Tabs>

## Client usage

`TalakWeb3Client` calls paths under `/auth/*`. When the handler is mounted at `/api/auth`, set `baseUrl: "/api"`:

```ts title="lib/auth-client.ts"
import { TalakWeb3Client } from "talak-web3";

export const authClient = new TalakWeb3Client({ baseUrl: "/api" });
```

```tsx title="components/sign-in.tsx"
"use client";

const { nonce } = await authClient.getNonce(address);
// sign SIWE message with wallet, then:
await authClient.loginWithSiwe(message, signature);
```

With `nextCookies()`, use `credentials: "include"` on `fetch` calls to `/api/auth/*`.

## Production: Redis Store

For production, use Redis-backed stores in `talak.config.ts`:

```ts title="talak.config.ts"
import { talakWeb3 } from "talak-web3";
import { RedisNonceStore, RedisRefreshStore, RedisRevocationStore } from "@talak-web3/auth/stores";
import Redis from "ioredis";

const redis = new Redis(process.env.REDIS_URL!);

export const app = talakWeb3({
  auth: {
    nonceStore: new RedisNonceStore(redis),
    refreshStore: new RedisRefreshStore(redis),
    revocationStore: new RedisRevocationStore(redis),
    accessTtlSeconds: 900,
    refreshTtlSeconds: 604_800,
  },
  chains: [
    /* ... */
  ],
});
```

<Callout type="warning">
  Redis is required for production deployments to ensure atomic nonce consumption and horizontal
  scaling.
</Callout>
